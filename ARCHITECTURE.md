# AR Снеговик — Полная Техническая Архитектура

## ЧАСТЬ 1. Сравнение подходов

### A. WebAR через браузер

| Параметр | Описание |
|---|---|
| Запуск после QR | URL → браузер → сайт |
| Камера | `getUserMedia` API |
| 3D | Three.js / Babylon.js поверх canvas |
| Взаимодействия | Touch events + Raycasting |
| iPhone | Safari ограничивает WebXR; Camera API работает; нет Surface Detection |
| Android | Chrome поддерживает WebXR; `getUserMedia` работает |
| Детский UX | ✅ Мгновенный старт, нет установки |
| Сложность разработки | Средняя |
| Сложность поддержки | Низкая–средняя |
| Сроки MVP | 2–4 недели |
| Риски UX | Отказ в разрешении камеры, нестабильный Safari |
| Оценка для MVP | ⭐⭐⭐⭐ |

**Ограничения:**
- iOS Safari: нет WebXR, нет стабильного Surface Detection
- AR.js (marker-based) работает, но требует маркер на бумаге
- 8th Wall / Zappar — платные (~$400/мес), дают стабильный World-AR
- Бесплатный вариант = "камера как фон + 3D поверх" (не World-locked)

---

### B. Нативное мобильное приложение

| Параметр | Описание |
|---|---|
| Запуск после QR | QR → App Store/Play Store → установка → открытие deep link |
| Камера | ARKit (iOS) / ARCore (Android) — полноценный AR |
| 3D | Unity AR Foundation / Native SceneKit/ARKit |
| Взаимодействия | Полный набор: жесты, face tracking, plane detection |
| iPhone | ✅ ARKit — лучший AR на рынке |
| Android | ✅ ARCore — стабильный, широкая поддержка |
| Детский UX | ❌ Нужна установка (~3–5 мин, барьер для ребёнка) |
| Сложность разработки | Высокая (Unity + iOS + Android + сертификаты) |
| Сложность поддержки | Высокая (App Store review, обновления) |
| Сроки MVP | 6–12 недель |
| Риски UX | Установка отпугивает; родители должны одобрить |
| Оценка для MVP | ⭐⭐ (но ⭐⭐⭐⭐⭐ для production) |

### C. Гибридный вариант

| Параметр | Описание |
|---|---|
| Запуск | QR → WebAR сразу работает как fallback |
| Улучшенный путь | Предложение установить приложение через deep link |
| Камера | Web: getUserMedia; App: ARKit/ARCore |
| 3D | Web: Three.js; App: Unity |
| Взаимодействия | Базовые в вебе, расширенные в приложении |
| UX | ✅ Работает для всех, лучше для тех кто установил |
| Сложность | Высокая (поддержка двух платформ) |
| Сроки | 8–14 недель |
| Оценка | ⭐⭐⭐ (переусложнён для MVP) |

---

### Итоговый выбор

| Этап | Рекомендация | Причина |
|---|---|---|
| **PoC/MVP** | **WebAR (вариант A)** | Нет установки, быстрый старт, работает сразу по QR |
| **Production** | **Нативное приложение (Unity)** | Настоящий AR, ARKit/ARCore, лучший UX |
| **Миграционный путь** | Web MVP → собрать метрики → портировать в Unity | Снижает риск, валидирует механики |

### Поток данных: QR → Взаимодействие со снеговиком

```
1. Ребёнок сканирует QR → браузер открывает URL
2. Загружается HTML/CSS/JS (< 200 KB gzip)
3. Three.js + модули загружаются с CDN
4. Показывается splash-экран "Привет, я Снеговик!"
5. Пользователь нажимает "Начать"
6. Запрос разрешения на камеру
7. getUserMedia() → видео-поток → <video> как фон
8. Three.js рендерит поверх: снеговик в сцене
9. API запрос: GET /api/quests → конфиг квестов
10. Показывается первый квест
11. Ребёнок взаимодействует → touch event → raycasting
12. Квест выполнен → анимация + звук + очки
13. localStorage.setItem('score', totalScore)
14. POST /api/score → сохранить результат
15. Финальный экран "Молодец!"
```

### Модули системы

| Модуль | Ответственность |
|---|---|
| `CameraManager` | getUserMedia, настройка задней камеры, fallback |
| `SceneManager` | Three.js сцена, освещение, рендер-луп |
| `Snowman` | 3D персонаж (геометрия или GLTF), анимации |
| `QuestManager` | Очередь квестов, проверка триггеров, очки |
| `UIManager` | Экраны, оверлеи, HUD, UI-анимации |
| `AudioManager` | Web Audio API, речевые реплики, звуки |
| `StateMachine` | Состояния игры, переходы |
| `ProgressStore` | localStorage + API синхронизация |
| `AnalyticsClient` | Отправка событий (GA4/PostHog) |

## ЧАСТЬ 3. MVP Flow

### State Machine

```
LOADING → START → CAMERA_PROMPT → AR_INIT → AR_ACTIVE
                                    ↓              ↓
                                  ERROR       QUEST_ACTIVE
                                  FALLBACK        ↓
                                           QUEST_COMPLETE
                                                  ↓
                                           ALL_QUESTS_DONE
```

### User Flow экран за экраном

| Экран | Триггер входа | Элементы | Действие |
|---|---|---|---|
| **Загрузка** | Открытие URL | Снежинка, "Загружаю..." | Авто |
| **Старт** | Загрузка завершена | ⛄ Лого, кнопка "Начать" | Нажать кнопку |
| **Запрос камеры** | Нажат "Начать" | Иконка камеры, объяснение, кнопка | Разрешить камеру |
| **AR сцена** | Камера открыта | Снеговик, HUD, кнопки | Взаимодействие |
| **Квест активен** | Автоматически | Панель квеста, подсказка | Выполнить задание |
| **Квест выполнен** | Триггер сработал | Анимация + очки | Нажать "Дальше" |
| **Финал** | Все квесты done | Трофей, счёт, "Молодец!" | Играть снова |
| **Ошибка** | Камера отказана | Иконка + объяснение | Повторить |
| **Fallback** | Нет камеры | 2D снеговик, базовые кнопки | Играть без AR |

### Ошибки и Fallback

| Ошибка | Что показать | Действие |
|---|---|---|
| Камера запрещена (`NotAllowedError`) | Инструкция по разрешению | Кнопка "Попробовать снова" |
| Камера недоступна (`NotFoundError`) | "Нет камеры" | Fallback режим |
| Нет HTTPS | "Нужен HTTPS" | Redirect на HTTPS |
| Slow network | Загрузка > 10s | Показать прогресс |
| WebGL не поддерживается | 2D fallback | PNG-анимация снеговика |
| iOS Safari ограничения | Предложить Chrome | Ссылка на Chrome |

## ЧАСТЬ 4. Технологический стек

### Стек для быстрого MVP (WebAR)

| Слой | Технология | Почему |
|---|---|---|
| **Камера** | `navigator.mediaDevices.getUserMedia` | Нативный браузерный API, работает везде |
| **3D/AR** | Three.js r160 | Зрелая библиотека, огромная экосистема, CDN |
| **Анимации 3D** | Three.js AnimationMixer + procedural | Не нужен GLTF-файл для MVP |
| **3D модель** | Процедурная геометрия → потом GLB | Нет зависимости от художника для PoC |
| **UI** | HTML/CSS/JS (нет фреймворка) | Минимум зависимостей, быстрая загрузка |
| **Звук** | Web Audio API | Встроен в браузер, не нужна библиотека |
| **Квесты** | Vanilla JS + JSON конфиг | Просто, понятно, легко расширять |
| **Хранение прогресса** | `localStorage` | Нет backend для MVP |
| **Хостинг** | Vercel / Netlify | Бесплатно, HTTPS по умолчанию |
| **Сборка** | Нет (ES Modules + importmap) | Работает без Node.js/Vite |

### Стек для Production

| Слой | Технология | Почему |
|---|---|---|
| **AR** | Unity + AR Foundation (ARKit/ARCore) | Нативный AR, plane detection, lighting estimation |
| **3D** | Unity HDRP / URP | Фотореалистичный или стилизованный рендер |
| **Персонаж** | Humanoid Rig + Animator | Профессиональные анимации |
| **UI** | Unity UI Toolkit | Адаптивный UI для мобильных |
| **Квесты** | ScriptableObjects + JSON | Гибкая конфигурация |
| **Backend** | Node.js + Fastify + PostgreSQL | Быстрый, типизированный |
| **CMS** | Strapi / Directus | Контент-менеджмент для квестов |
| **Analytics** | PostHog (self-hosted) | Приватность, COPPA compliance |
| **CI/CD** | GitHub Actions + Fastlane | Автосборка iOS/Android |
| **CDN** | Cloudflare | Глобальный кэш для ассетов |

## ЧАСТЬ 7. Требования к 3D-модели

### Технические параметры

| Параметр | MVP (процедурная) | Production |
|---|---|---|
| Формат | Three.js primitives | GLB (GLTF 2.0) |
| Полигонаж | ~2 000 poly | 5 000–15 000 poly |
| LOD | Нет | LOD0: 15k, LOD1: 5k, LOD2: 1k |
| Текстуры | MeshToonMaterial | PBR: Albedo, Normal, Roughness 1024×1024 |
| Rig | Нет | Humanoid + custom bones (нос, шарф) |
| Анимации | Procedural JS | Keyframe, встроены в GLB |
| Вес файла | 0 KB | < 3 MB с анимациями |
| Оптимизация | — | Draco compression, меш merging |

### Обязательные анимации

| Анимация | Длина | Описание |
|---|---|---|
| `idle` | ∞ loop | Лёгкое покачивание, моргание |
| `dance` | 4–6 сек | Ритмичные движения, руки вверх |
| `wave` | 2–3 сек | Махание правой рукой |
| `sing` | 4–8 сек | Рот открывается, голова качается |
| `happy` | 2–3 сек | Прыжки, руки вверх |
| `surprised` | 1–2 сек | Резкий наклон назад, руки в стороны |
| `win` | 3–4 сек | Победный танец, конфетти |
| `lose` | 2–3 сек | Грустно опускает голову |
| `tap_nose` | 0.5 сек | Нос светится, моргание |
| `nod` | 1.5 сек | Кивание головой |

### Требования к аудио

| Файл | Формат | Длина | Размер |
|---|---|---|---|
| Фразы (15–20 фраз) | MP3/OGG, 44kHz | 1–3 сек | < 100 KB каждый |
| Смех | MP3 | 2–3 сек | < 150 KB |
| Музыкальная тема (idle) | MP3 | 30–60 сек loop | < 500 KB |
| Звук победы | MP3 | 3–5 сек | < 200 KB |
| Звук тапа | MP3 | 0.2–0.5 сек | < 30 KB |
| Звук квеста | MP3 | 1–2 сек | < 80 KB |

**Важно:** Autoplay заблокирован браузерами. Первый звук ТОЛЬКО после пользовательского взаимодействия (tap/click).

## ЧАСТЬ 9. Реальные ограничения

### Что реально работает в браузере

| Функция | Статус | Примечание |
|---|---|---|
| Запрос камеры | ✅ | getUserMedia, нужно HTTPS |
| Видео-фон | ✅ | `<video>` за canvas |
| 3D рендеринг | ✅ | Three.js WebGL |
| Touch-взаимодействие | ✅ | touchend events |
| Web Audio | ✅ | Только после тапа |
| localStorage | ✅ | Без backend |
| Orientation | ✅ | DeviceOrientationEvent (нужно разрешение iOS 13+) |

### Что нестабильно на iPhone

| Проблема | Серьёзность | Обходной путь |
|---|---|---|
| WebXR не поддерживается Safari | 🔴 Критично | Нет нативного AR без приложения |
| Autoplay видео/аудио | 🔴 Критично | Требовать первый тап от пользователя |
| DeviceOrientation нужно разрешение | 🟡 Среднее | Кнопка "Разрешить ориентацию" |
| Видео не запускается без `playsinline` | 🟡 Среднее | Всегда добавлять атрибут |
| PWA ограничения в Safari | 🟡 Среднее | Нет фонового аудио |
| iframe getUserMedia | 🔴 Критично | Не использовать iframe |

### Что лучше работает только в приложении

- Plane detection (определение горизонтальных поверхностей)
- World-locked объекты (снеговик стоит на полу, не "плавает")
- Face tracking, hand tracking
- Стабильная ориентация без дрожания
- Офлайн-работа
- Push уведомления

### Почему нельзя обещать "идеальный AR без установки"

1. **iOS Safari не поддерживает WebXR** — это принципиальное ограничение Apple
2. **getUserMedia без HTTPS не работает** — только localhost или HTTPS
3. **Surface Detection** в вебе нестабилен даже с WebXR
4. **Без разрешения камеры** приложение не работает — нужен понятный UX для запроса
5. **Autoplay блокируется** — первый звук только после пользовательского действия
6. **Слабые устройства** (старые Android) могут дать < 20 FPS на Three.js

## ЧАСТЬ 10. Roadmap

### Этап 1 — PoC (1–2 недели)

| Параметр | Описание |
|---|---|
| **Результат** | Веб-страница: камера + 3D снеговик + 3 анимации + 1 квест |
| **Команда** | 1 frontend разработчик |
| **Риски** | iOS Safari ограничения, нет художника для 3D |
| **Критерии готовности** | Работает на iPhone Safari + Chrome Android, снеговик отображается |

### Этап 2 — MVP (3–6 недель)

| Параметр | Описание |
|---|---|
| **Результат** | 5 квестов, система очков, финальный экран, QR для тестирования |
| **Команда** | Frontend, 3D-художник, QA |
| **Риски** | Производительность на слабых Android, нет fallback для старых браузеров |
| **Критерии готовности** | Тест с 10+ детьми 4–8 лет, 80% завершают все квесты |

### Этап 3 — Production (8–16 недель)

| Параметр | Описание |
|---|---|
| **Результат** | Unity приложение (iOS + Android), полноценный AR, 10 квестов, CMS |
| **Команда** | Unity dev, iOS/Android dev, backend, 3D-художник, звукорежиссёр, QA |
| **Риски** | App Store review, Apple-сертификаты, ARCore совместимость |
| **Критерии готовности** | App Store публикация, 4.5+ рейтинг в отзывах |

### Этап 4 — Масштабирование

| Параметр | Описание |
|---|---|
| **Результат** | Мульти-персонажи, сезонные квесты, CMS для контент-менеджеров, аналитика |
| **Команда** | Полная команда + product manager + дизайнер |
| **Риски** | COPPA compliance (дети < 13 лет), обработка персданных |
| **Критерии готовности** | 10 000+ MAU, NPS > 70 |

### Миграционный путь: Web → Unity

```
Этап 1: WebAR MVP → валидация механик
Этап 2: Параллельная разработка Unity
         • Портирование квест-логики из JSON (единый формат)
         • Использование тех же 3D-моделей (GLB → Unity)
         • Единый backend API
Этап 3: Unity app release → web остаётся как fallback
Этап 4: Постепенный переход трафика на native app
```
